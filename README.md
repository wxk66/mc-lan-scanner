# 我的世界局域网扫描器 (MC LAN Scanner)

扫描局域网 / Radmin VPN 等虚拟局域网中的 Minecraft (Java 版) 服务器，自动识别 MOTD、版本、在线人数、玩家列表和延迟。提供命令行和图形界面两种用法。

> 起因：用 Radmin VPN 联机时，队友的服务器 IP 散落在 `26.x.x.x` 各个网段，端口也五花八门（不一定是 25565）。手动一个个试太麻烦，于是做了这个工具。

![界面预览](docs/screenshot.png)

## 功能特性

- **三种扫描模式**
  - **监听局域网广播**（推荐）：被动监听"对局域网开放"世界发出的多播广播（`224.0.2.60:4445`），无需猜 IP 和端口，开了局域网世界的服务器会主动报告自己。
  - **主动扫描指定范围**：对指定的 IP 段 + 端口段逐个探测，适合查找不广播的专用服务器。
  - **自动检测本机网段**：自动识别本机所有网卡（含 Radmin VPN 的 `26.x.x.x`），扫描所在网段。
- **真实协议探测**：使用 Minecraft 服务器列表 Ping (SLP) 协议确认，而非简单的端口扫描，能拿到真实的 MOTD、服务端类型与版本、在线/最大人数、在线玩家名。
- **乱码修复**：自动修复部分老服务器（如 1.12.2 + 某些插件）的"双重/多重编码"乱码 MOTD，把 `ÂÂâ欢...` 这类还原成正常中文。
- **延迟检测**：用 TCP 握手往返耗时测量延迟（比 ICMP ping 更可靠，很多 VPN 主机禁 ping），以信号条 + 毫秒数直观显示。
- **彩色卡片界面**：每个服务器一张卡片，颜色区分版本、在线状态、延迟高低；地址一键复制。
- **结果导出**：支持导出 JSON / 纯文本。
- **高 DPI 适配**：在 150% / 200% 缩放的高分屏上清晰显示。
- **零第三方依赖**：仅用 Python 标准库（含 tkinter），兼容 Python 3.8+。

## 快速开始

### 方式一：直接运行图形界面

```bash
python mc_gui.py
```

打开后默认是"监听局域网广播"模式，点击蓝色「开始扫描」按钮即可。

### 方式二：命令行

```bash
# 默认：监听局域网广播（最省事）
python mc_scanner.py

# 监听更久（默认 15 秒）
python mc_scanner.py --lan-time 30

# 主动扫描指定网段和端口
python mc_scanner.py -t 26.0.0.0/24 -p 25565-25600

# 指定多个目标和端口
python mc_scanner.py -t 192.168.1.0/24,26.158.0.1-26.158.0.50 -p 25565,25566,30000-30010

# 自动检测本机网段并扫描
python mc_scanner.py --scan

# 导出结果（按扩展名自动判断格式）
python mc_scanner.py -o result.json
python mc_scanner.py -o result.txt
```

### 方式三：使用打包好的 exe（Windows，无需装 Python）

从 [Releases](../../releases) 下载 `MC局域网扫描器.exe`，双击运行。

## 命令行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-t, --targets` | 扫描目标（逗号分隔）：IP / CIDR / 范围。留空则不主动扫描 | 无 |
| `-p, --ports` | 端口（逗号分隔，支持范围） | `25565-25600` |
| `--scan` | 自动检测本机网段并主动扫描 | 关 |
| `--lan` | 主动扫描的同时监听局域网广播 | 关 |
| `--lan-time` | 局域网广播监听时长（秒） | `15` |
| `--timeout` | 单个连接超时秒数 | `1.0` |
| `--workers` | 并发线程数 | `200` |
| `-o, --output` | 导出结果到文件（`.json` / `.txt`） | 无 |
| `--format` | 强制导出格式（`json` / `txt`），覆盖按扩展名判断 | 无 |

无参数运行时，仅监听局域网广播。

## 工作原理

- **SLP 协议**：连接服务器端口后发送握手包（next-state = 1）和状态请求包，服务器返回包含 MOTD / 版本 / 人数的 JSON。详见 [wiki.vg - Server List Ping](https://wiki.vg/Server_List_Ping)。
- **局域网广播**：Minecraft 的"对局域网开放"功能会每隔 1.5 秒向多播组 `224.0.2.60:4445` 发送 `[MOTD]...[/MOTD][AD]端口[/AD]` 格式的 UDP 包。本工具加入该多播组监听，再对发现的地址做一次 SLP 探测补全详细信息。
- **延迟测量**：对已知可达的端口做多次 TCP 三次握手，取往返耗时最小值，避免 ICMP 被防火墙拦截的问题。

## 常见问题

**Q：协议号显示 47 但版本写着 1.21？**
A：装了 ViaVersion 等多版本兼容插件的服务器，对未知协议握手会返回兜底值 47（1.8 的协议号），MOTD 里的版本名才是真实的。

**Q：最大人数显示 ∞？**
A：部分服务器（如 2b2t 类）故意把最大人数设为负数表示"无上限"，工具将其美化为 ∞。

**Q：某个服务器广播了但探测失败？**
A：可能是它的防火墙拦截了非 MC 客户端的连接，或它是特殊代理。这种情况会保留广播里的 MOTD 并标注"探测失败"。

**Q：Radmin 网段太大扫不完？**
A：Radmin 的网卡掩码通常是 `/8`（1600 多万地址），自动模式会收窄为本机所在的 `/24`。最可靠的做法是直接在 Radmin 客户端里看到队友 IP，用 `-t` 指定。或者干脆用默认的广播监听模式。
